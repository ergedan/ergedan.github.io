一、Logstash

开源的服务器端数据处理管道，能够同时从多个来源采集数据、转换数据，然后将数据发送到指定的存储库中。

1、Logstash功能

  • （1）即插即用
  • （2）可扩展，Logstash采用可插拔框架。
  • （3）可靠性，如果Logstash节点发生故障，Logstash会通过持久化队列来保证运行中的事件至少一次被送达。
  • （4）监控，全方位监视每个节点的部署

（1）Logstash目录介绍

位置 说明
/usr/share/logstash logstash安装的主目录
/usr/share/logstash/bin 二进制脚本位置
/etc/logstash 配置文件存放位置
/etc/logstash/conf.d logstash管道配置文件，注意只要文件放置在此目录下都生效
/var/log/logstash 日志位置
/usr/share/logstash/plugins 插件位置
/var/lib/logstash 数据存放位置

（2）/etc/logstash/logstash.yml配置文件

选项 说明
node.name 节点名称
path.data 数据路径
pipeline.workers 工作的并行数量
pipeline.batch.size 单工作线程最大事件数量
path.config 配置文件路径
config.reload.automatic 设置为true定期检查是否已更改
config.reload.interval 检查配置文件更改的间隔时长
config.debug 是否显示调试信息
http.host 绑定的IP地址
http.port 绑定的端口号
log.level 日志级别
log.format 日志格式
path.logs 日志路径
path.plugins 插件位置

3、logstash修改配置文件

[root@node4 ~]# grep '^[^#]' /etc/logstash/logstash.yml

4、输入插件

（1）file插件

选型 输入类型
path array
start_position string，one of[beginning,end]
exclude 处理某种类型的文件
max_open_files 打开的最大文件数
stat_interval 间隔时长

示例：

1 input {
2         file {
3                 path    => "/root/test"
4                 start_position  => beginning
5         }
6 }

（2）exec

选项 类型
command string
interval number

示例：

1 input {
2         exec {
3                 command => "ls -s /etc/fstab"
4                 interval        => 5
5         }
6 }

（3）http插件

选项 类型
host string
password password
port number
response_headers hash
user string
ssl boolean

示例：

1 input {
2         http {
3                 host => "127.0.0.1"
4                 port    => "80"
5                 codec   => "plain"
6         }
7 }

（4）tcp插件

选项 类型
port number
host string
mode string，one of [server,client]
proxy_protocol boolean

示例：

1 input {
2         tcp {
3                 port    => 80
4                 mode    => "server"
5                 ssl_enable      => false
6         }
7 }

（5）redis插件

选项 说明
data_type string，one of[list，channel，pattern_channel]
db number，默认为0
host string
password password
port number

timeout|number
|key|stirng

示例：

1 input {
2         redis {
3                 data_type       => "list"
4                 port    => "6379"
5                 db      => "0"
6                 password        => "centos"
7                 host    => "172.18.4.66"
8         }
9 }

19         }

6、过滤插件

（1）grok插件

通过将文本模式组合成与日志匹配的东西来工作。

grok模式的语法是：%{SYNTAX:SEMANTIC}

  • 选项
    match，默认值为{}

    filter {

    match => {
            "message" => "%{COMBINEDAPCHEDLOG}"
}

    }

removed_field，默认值为[]

示例：

 4 filter {
 5         grok {
 6                 match => {
 7                         "message" => "%{COMBINEDAPACHELOG}"
 8                 }
 9                 remove_field    => "message"
10         }
11 }

（2）geoip插件

GeoIP过滤器来自Maxmind GeoLite2数据库添加有关的IP地址的地址位置的信息。

选项 说明
source 数据来源
target 指定logstash应该存储geoip数据的字段，默认为geoip
database 存放数据库的路径

下载地址：https://dev.maxmind.com/zh-hans/geoip/geoip2/geolite2-%E5%BC%80%E6%BA%90%E6%95%B0%E6%8D%AE%E5%BA%93/

如果我们需要使用GeoIP插件需要下载数据库

示例：
[root@node4 logstash]#gzip -d GeoLite2-City.mmdb.gz #加压数据库到当前目录下

 4 filter {
 5         grok {
 6                 match => {
 7                         "message" => "%{COMBINEDAPACHELOG}"
 8                 }
 9                 remove_field    => "message"
10         }
11         geoip {
12                 sourece => "clientip"
13                 target  => "geoip"
14                 database        => "/etc/logstash/Geolite2-City.mmdb"
15         }
16 }

（3）mutate

mutate筛选器可以重命名、删除、替换和修改事件中的字段。

选项 说明
gsub 仅对字符串字段有效
split 分隔
join 仅对数组类型有效
merge 合并两个数组或哈希字段
rename 重命名
update 更新某个字段
replace 替代某个字段

7、编码插件（codec）

采用JSON编码

直接输入预定义好的JSON数据，这样就可以省略掉fileter/grok配置

注意：
Nginx作为一个代理服务器运行的话，访问日志里有些变量，如$upstream_response_time，可能不是一个数据，可能是一个“-”字符串，这会直接导致logstash对输入数据验证报异常。

使用sed在输入之前先替换“-”为0

二、示例

1、把http日志发送到elasticsearch

 1 input {
 2         file {
 3                 path    => "/var/log/httpd/access_log*"
 4                 start_position  => beginning
 5         }
 6 }
 7 filter {
 8         grok {
 9                 match   => {
10                         "message"       => "%{COMBINEDAPACHELOG}"
11                 }
12         }
13         geoip {
14                 source  => "clientip"
15                 database        => "/etc/logstash/GeoLite2-City.mmdb"
16                 target  => "geoip"
17         }
18 }
19 output {
20         elasticsearch {
21                 hosts   => ["http://192.168.4.61:9200/","http://192.168.4.62:9200/","http://192.
   168.4.63:9200/"]
22                 index   => "logstash-%{+YYYY.MM.dd}"
23                 action  => "index"
24                 document_type   => "apache_log"
25         }
26 }
