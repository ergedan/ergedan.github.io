HTTP正向和反向代理

发表于 2017-12-09 | 分类于 服务 ， Apache ， HTTP正向和反向代理 | 阅读数
字数统计: 1,071 字 | 阅读时长 ≈ 5 分钟

一、正向代理

1、概念

正向代理是一个位于客户端和目标服务器之间的代理服务器。为了从原始服务器取得内容，客户端向代理服务器发送一个请求，并且指定代理服务器向目标服务器转交并获得内容返回给客户端。

2、正向代理服务器作用

  • 访问无法访问的服务器B
  • 加速访问服务器B
  • Cache作用
  • 隐藏访问者行踪

二、反向代理

1、概念

反向代理是为服务端转发请求，客户端将请求发送至反向代理服务器，反向代理服务器再将服务器请求转发给真正的服务器进行处理请求，响应时后端真正的服务器将处理结果发送给反向代理，再由反向代理构建响应并响应给客户端。

2 、反向代理作用

  • 保护和隐藏原始资源服务器
  • 负载均衡作用
  • 加密和SSL加速
  • 缓存静态内容
  • 压缩功能

三、HTTP正向代理

1、ProxyRequest指令

Description:    Enables forward (standard) proxy requests
Syntax:    ProxyRequests On|Off
Default:    ProxyRequests Off
Context:    server config, virtual host
Status:    Extension
Module:    mod_proxy

2、ProxyVia指令

是否在响应首部添加Via字段

Description:    Information provided in the Via HTTP response header for proxied requests
Syntax:    ProxyVia On|Off|Full|Block
Default:    ProxyVia Off
Context:    server config, virtual host
Status:    Extension
Module:    mod_proxy

off：将不会采取特殊处理
on：每个请求和应答都会对应当前主机得到一个“Via：”头
full：每个产生的“Via：”头都会额外加入Apache服务器的版本
block：每个代理请求中的所有”Via：”头行都将被删除

示例：

 1 <VirtualHost *:80>
 2         ServerName www.test.com
 3         DocumentRoot "/app/web/test"
 4         ProxyRequests ON
 5         ProxyVia ON
 6         <Directory "/app/web/test">
 7                 Require all granted
 8         </Directory>
 9         <Proxy *>
10                 Require all granted
11         </Proxy>
12 </VirtualHost>

四、HTTP反向代理

1、Proxy指令

Description:    Container for directives applied to proxied resources
Syntax:    <Proxy wildcard-url> ...</Proxy>
Context:    server config, virtual host
Status:    Extension
Module:    mod_proxy

2、ProxyMatch指令

Description:    Container for directives applied to regular-expression-matched proxied resources
Syntax:    <ProxyMatch regex> ...</ProxyMatch>
Context:    server config, virtual host
Status:    Extension
Module:    mod_proxy

可以使用正则表达式。

3、ProxyPass指令

此指令可以防止在location容器中使用。

Description:    Maps remote servers into the local server URL-space
Syntax:    ProxyPass [path] !|url [key=value [key=value ...]] [nocanon] [interpolate] [noquery]
Context:    server config, virtual host, directory
Status:    Extension
Module:    mod_proxy
Compatibility:    Unix Domain Socket (UDS) support added in 2.4.7

注意：如果第一个参数加了“/”，则URL部分也必须加“/”.

选项：

keepalive=off|on：设置后端是否发开启长连接
lbset=N：设置优先级
ping=N：检查将康状态的时间间隔
retry=N：尝试时长
status=VALUE：将节点设置为何种状态

    D：禁用，不再接受任何请求
    S：停止
    I：忽视状态
    H：备份节点
    E：错误状态
    N：drain模式
timeout：超时时长

4、ProxyPassMatch指令

Description:    Maps remote servers into the local server URL-space using regular expressions
Syntax:    ProxyPassMatch [regex] !|url [key=value [key=value ...]]
Context:    server config, virtual host, directory
Status:    Extension
Module:    mod_proxy

5、ProxyPassReverse指令

Description:    Adjusts the URL in HTTP response headers sent from a reverse proxied server
Syntax:    ProxyPassReverse [path] url [interpolate]
Context:    server config, virtual host, directory
Status:    Extension
Module:    mod_proxy

6、ProxyPreserveHost指令

Description:    Use incoming Host HTTP request header for proxy request
Syntax:    ProxyPreserveHost On|Off
Default:    ProxyPreserveHost Off
Context:    server config, virtual host, directory
Status:    Extension
Module:    mod_proxy
Compatibility:    Usable in directory context in 2.3.3 and later.

7、ProxyStatus指令

决定是否开启proxy状态信息

Description:    Show Proxy LoadBalancer status in mod_status
Syntax:    ProxyStatus Off|On|Full
Default:    ProxyStatus Off
Context:    server config, virtual host
Status:    Extension
Module:    mod_proxy
Compatibility:    Available in version 2.2 and later

8、BalancerMember指令

指定负载均衡成员

Description:    Add a member to a load balancing group
Syntax:    BalancerMember [balancerurl] url [key=value [key=value ...]]
Context:    directory
Status:    Extension
Module:    mod_proxy
Compatibility:    BalancerMember is only available in Apache HTTP Server 2.2 and later.

9、ProxySet

Description:    Set various Proxy balancer or member parameters
Syntax:    ProxySet url key=value [key=value ...]
Context:    directory
Status:    Extension
Module:    mod_proxy
Compatibility:    ProxySet is only available in Apache HTTP Server 2.2 and later.

（1）http均衡算法

byrequests：默认算法，基于请求数量计算权重
bytraffic：基于I/O流量大小计算权重
bybusyness：基于挂起的请求数量计算权重

示例：

 1 <VirtualHost *:80>
 2         ServerName 192.168.4.61
 3         DocumentRoot "/app/web/test"
 4         ProxyRequests OFF
 5         ProxyPass "/" "balancer://http"
 6         ProxyPassReverse "/" "balancer://http"
 7         <Directory "/app/web/test">
 8                 Require all granted
 9         </Directory>
10         <Proxy balancer://http>
11                 BalancerMember http://192.168.4.62
12                 BalancerMember http://192.168.4.63 
13                 ProxySet lbmethod=byrequests
14         </Proxy>
15 </VirtualHost>
