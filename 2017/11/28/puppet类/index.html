类

Class是用于通用目标或目的的一组资源，因此，它是命名的代码块，在某未知创建之后可在puppet全局使用。类似于其他编程语言中类的功能，puppet的类可以继承，也可以包含子类。

格式：

class my_class {
    ...puppet code ...
}

注意：类的名称只能以小写字母开头，可以包含小写字母、或下划线。另外每个类都会引入一个新的变量scope，这意味着在任何时候访问类中的变量时，都得使用期完全限定的名称。不过在本地scope可以重新为top scope 中的变量赋予一个新值。

（1）类声明

  • 方式一

    include CLASS_NAME1 ,CLASS_NAME2…

  • 方式二
    #
    class {‘CLASS_NAME’:
              attribute   => value,
          ... 
  }

示例：

  1 class apache2 {
  2         $webpkg = $operationsystem ? {
  3                 /(?i-mx:(centos|redhat|fedora))/        => 'httpd',
  4                 /(?i-mx:(ubuntu|debian))/       => 'apache2',
  5                 default => 'httpd',
  6         }
  7         package{'$webpkg':
  8                 ensure  => latest,
  9         }
 10 }

类继承方式

格式：

class SUB_CLASS_NAME inherits PARENT_CLASS_NAME {
    ...puppet code
}

示例：

  1 class apache2 {
  2         $webpkg = $operationsystem ? {
  3                 /(?i-mx:(centos|redhat|fedora))/        => 'httpd',
  4                 /(?i-mx:(ubuntu|debian))/       => 'apache2',
  5                 default => 'httpd',
  6         }
  7         package{'$webpkg':
  8                 ensure  => latest,
  9         }
 10 }
 11 class apache2::web inherits apache2 {
 12         file{'file':
 13                 path    => '/etc/httpd/conf.d/http-v.conf',
 14                 ensure  => file,
 15                 source  => 'root/http-v.conf',
 16         }
 17 }

在子类中为父类的资源新增或覆盖指定的属性的值

格式：

Type['tile'] {
    attribute => value,
    ...
}

在子类中为父类的资源的某属性增加心智

Type['title'] {
    attribute +> value,
    ...
}

带参数的类

在定义在带参数的类时，需要将参数声明在类名后的小括号“（）”，参数可以有默认值，如果使用多个参数，彼此间要使用逗号分隔。在类的内部使用参数的方式同使用本地变量。

格式：

    class mysql ($user = 'mysql', $port = 3306) {
                ...
     }

向类传递参数时，其方式如同定义资源的属性。
格式：

class http($pkgname='apache2') {
    package{'$pkgname':
        ensure  => latest,
    }
}

5、puppet模板

puppet模板时erb（embedded ruby）格式的。

文本文件中内嵌变量替换机制：
<%= @VARIABLE_NAME %>

格式：

file{‘title’:
ensure => file,
path =>
content => template(‘/path/to/ERB_FILE’)
}
