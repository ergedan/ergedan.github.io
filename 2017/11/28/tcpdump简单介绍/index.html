tcpdump简单介绍

发表于 2017-11-28 | 分类于 命令 ， tcpdump简单介绍 |

tcpdump是一个运行在命令行下的嗅探工具，它允许用户拦截和显示发送或收到过网络连接到该计算机的TCP/IP和其他数据包。在linux中需要使用libpcap捕捉数据的库。

1、tcpdump命令

SYNOPSIS

tcpdump [ -AbdDefhHIJKlLnNOpqStuUvxX# ] [ -B buffer_size ]
        [ -c count ]
        [ -C file_size ] [ -G rotate_seconds ] [ -F file ]
        [ -i interface ] [ -j tstamp_type ] [ -m module ] [ -M secret ]
        [ --number ] [ -Q|-P in|out|inout ]
        [ -r file ] [ -V file ] [ -s snaplen ] [ -T type ] [ -w file ]
        [ -W filecount ]
        [ -E spi@ipaddr algo:secret,...  ]
        [ -y datalinktype ] [ -z postrotate-command ] [ -Z user ]
        [ --time-stamp-precision=tstamp_precision ]
        [ --immediate-mode ] [ --version ]
        [ expression ]

2、tcpdump选项类型

  • 控制tcpdump程序行为
  • 控制数据怎么样显示
  • 控制显示什么数据
  • 过滤命令

（1）控制程序行为

选项 说明
-w 直接将抓取的数据写入文件中
-r 读取抓取的文件
-c 设置捕捉报文数量，当达到数量时程序自动终止，无需使用ctrl+x或kill命令
-i 指定网络接口的数据包
-p 将网卡接口设置为非混杂模式
-s 控制数据的截取长度，默认只抓取68字节

（2）控制信息如何显示

选项 说明
-a 强制将网络地址显示为名称，默认显示方式
-n 阻止将地址显示为名字
-N 阻止将域名转化
-f 阻止远端名称解析

（3）控制显示什么数据

选项 说明
-v 将会打印TTL字段
-q 静默显示
-e 显示链路层头信息
-x 将报文以十六进制形式显示

静默显示

[root@node1~]#tcpdump -c1 -n -q host 192.168.4.61

（3）过滤

选项 说明
host 指定主机，可以通过名称或IP地址指定
src 指定数据流的来源
det 指定数据流的目的地
broadcast 指定广播地址
multicast 指定多播地址
IP、Appletalk、tcp、udp、icmp 指定抓取的协议
port 指定端口

2、常用的表达式

非：！或者not
且：&&或者and
或：||或者or

示例：

[root@node1~]#tcpdump -c1 port 22 and host 192.168.4.61

tcpdump: verbose output suppressed, use -v or -vv for full protocol decode
listening on eth0, link-type EN10MB (Ethernet), capture size 262144 bytes
12:00:13.938273 IP node1.ssh > 192.168.4.1.12340: Flags [P.], seq 2146131600:2146131840, ack 1282914057, win 314, length 240

[root@node1~]#tcpdump -c1 -nn -i eth0

tcpdump: verbose output suppressed, use -v or -vv for full protocol decode
listening on eth0, link-type EN10MB (Ethernet), capture size 262144 bytes
14:12:39.593263 IP 192.168.4.61.22 > 192.168.4.1.12340: Flags [P.], seq 2146665200:2146665440, ack 1282925385, win 314, length 240

    -i：指定接口
    -nn：表示IP和port方式显示包，不将IP转为主机名
