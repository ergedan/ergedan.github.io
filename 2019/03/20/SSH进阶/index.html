疑惑

最近在研究Jenkins，想到一个问题，如果我在同一台linux主机上管理多个项目，而且项目存放在github上的地址又不一样，以不同的用户去访问，如何去访问呢？

2、多个账户配置

在实际过程中，不同项目会有不同的账户去登录github等git服务器上。我们如何配置多个Git账号呢？

~]# ssh-keygen

Generating public/private rsa key pair.
Enter file in which to save the key (/root/.ssh/id_rsa): /root/.ssh/id_rsa_test        #如果有多个SSH用户，需要指定名称。
Enter passphrase (empty for no passphrase): 
Enter same passphrase again: 
Your identification has been saved in /root/.ssh/id_rsa_test.
Your public key has been saved in /root/.ssh/id_rsa_test.pub.
The key fingerprint is:
SHA256:0fNPZ/FJsD3fbUKa1MqU/itvst7vueMqaOTaf+79w6U root@master
The key's randomart image is:
+---[RSA 2048]----+
|             .   |
|         .   o+  |
|        . o +.o= |
|         . B =. O|
|        S   B o.O|
|         .   + =.|
|        o .   +..|
|        .+ .ooE=.|
|       .o..oBX*BO|
+----[SHA256]-----+

创建config文件

vim /root/.ssh/config

Host github
        HostName github.com
        User git
        PreferredAuthentications publickey
        IdentityFile /root/.ssh/id_rsa_test

#

Host     相当于定义别名
HostName     服务器地址
Port    指定端口号
User    登录的用户名
PreferredAuthentications     选择认证方式
IdentityFile    私钥文件的路径

3、代理登录

有时候我们无法直接连接到某台服务器，而需要一台中转服务器进行中转。

(1)netcat实现

使用nc命令实现。假设本地SSH代理服务端口为3000

示例

Host gateway
    HostName 127.0.0.1
    Port 3000
    User root

Host db
    HostName db.internal.example.com
    User root
    ProxyCommand nc -x gateway  %h %p

#
%h 表示目标地址
%p 表示目标端口

ProxyCommand ssh gateway nc %h %p

通过proxyCommand，可以在开启SSH之前执行一个命令打开代理隧道，nc %h %p 含义为在跳板机上使用nc开启了远程隧道。

（2）使用SSH自身代理功能

示例

Host B
    HostName %h
    User dsl
    Port 1046
    IdentityFile ~/.ssh/id_dsa

Host C
    HostName %h
    User dsl 
    Port 1046
    IdentityFile ~/.ssh/id_rsa
    ProxyCommand ssh -W %h:%p B

通过ProxyCommand命令先与gateway建立ssh连接，并把这个中间连接当做一个代理使用。

问题集合

如果执行ssh-add 命令提示如下错误：

Could not open a connection to your authentication agent.

解决方式

那么请执行eval $(ssh-agent)命令后再重试，如果还不行，请再执行ssh-agent bash命令后再执行eval $(ssh-agent)后执行添加命令。另外上述测试配置的命令不能执行时也可以在ssh-agent bash执行完后再测试
